This guide walks you through integrating Okto APIs into your application. You'll set up your developer account, obtain API keys, authenticate requests, and make API calls using the Okto Auth Token. Follow the steps below to get started.

<Steps>
  <Step>
  ### **Prerequisites**
  <CollapsibleCallout type="warning" title="Essential Setup Required">
    Before you begin, set up your [developer dashboard](https://dashboard.okto.tech) by making sure you have :
    - Created your [Okto Developer Dashboard](/docs/developer-admin-dashboard/dashboard-account) account
    - Get your [Okto credits](/docs/developer-admin-dashboard/okto-faucet) from the dashboard
    - Obtained your [API keys](/docs/developer-admin-dashboard/api-key) from the dashboard
    - Enabled the specific [Chains and Tokens](/docs/developer-admin-dashboard/wallet-control) you plan to use in your application
    - Optional: Enabled [Sponsorship](/docs/developer-admin-dashboard/sponsorship) for the desired chains

    If you need help, reach out to us on our <ExternalLink href="/docs/openapi/troubleshooting">troubleshooting form</ExternalLink> and we will contact you.
    </CollapsibleCallout>

- **Okto API Keys:** You need your `CLIENT_PRIVATE_KEY` and `CLIENT_SWA`. Obtain these from the [Okto Developer Dashboard](https://dashboard.okto.tech/login).
- **Okto Account Setup:** Ensure you have sufficient <ExternalLink href="https://dashboard.okto.tech/okto-credits">Okto Credits</ExternalLink>, enable the required <ExternalLink href="https://dashboard.okto.tech/wallet-controls">Chains and Tokens </ExternalLink>, and optionally <ExternalLink href="https://dashboard.okto.tech/paymaster">Activate Sponsorship</ExternalLink>.
- **Google OAuth Credentials (if using Google authentication):** Create OAuth 2.0 credentials in the <ExternalLink href="/docs/react-sdk/authenticate-users/google-oauth/google-console-setup">Google Cloud Console</ExternalLink> to get your `GOOGLE_CLIENT_ID`.

</Step>

<Step>

### **Authenticate**

Authentication with the Okto API requires generating an Okto Auth Token, which is used for all subsequent API calls.

- For testing purpose, Use the [Okto Auth Token Generator](/tools) to obtain an authentication token.
- Alternatively, implement authentication in your code using our [Sample Implementation](https://github.com/okto-hq/okto-sdkv2-ts-template/tree/main/src/api-template) to implement authentication in your code
- Ensure the session keys/Okto auth token are stored securely.

  <Callout title="Note" type="info">
    Each session is created for both the user and client, with a default
    validity of 10 days. You can customize this duration as needed.
  </Callout>

</Step>

<Step>

### **Make API calls**

To effectively interact with the Okto API, please check the [API Overview Guide](/docs/openapi). This will help you understand the API architecture, and the authentication flow.

After obtaining the Okto Auth Token, you can use it to send API requests:

- Include the token in the request headers as:

```bash
Authorization: Bearer <OKTO_AUTH_TOKEN>
```

- Check out our [API Documentation](/docs/openapi#explore-api) for a complete list of available endpoints, and start testing the API instantly using your generated Okto Auth Token.
- For a hands-on example, check out our [Template App](https://github.com/okto-hq/okto-sdkv2-ts-template/tree/main/src/api-template) to understand request structures and best practices.

</Step>
</Steps>

## API References

Okto's API landscape is composed of two distinct types of APIs:

### JSON-RPC Methods

JSON-RPC methods directly interact with the Okto Gateway and are primarily used for authentication and executing user operations (intents).

| **Name** | **API Endpoint** | **Headers** | **Description** |
|----------|------------------|-------------|-----------------|
| **Google Authentication** | <ExternalLink href="/docs/openapi/authenticate/google-oauth/loginUsingOktoAuth">`POST https://sandbox-okto-gateway.oktostage.com/rpc`</ExternalLink> | `Content-Type: application/json` | Handles user authentication via Google OAuth and session management. |
| **Token Transfer** | <ExternalLink href="/docs/openapi/tokenTransfer">`POST https://sandbox-okto-gateway.oktostage.com/rpc`</ExternalLink> | `Content-Type: application/json` `Authorization: Bearer <OKTO_AUTH_TOKEN>` | Handles token transfers between addresses. |
| **NFT Transfer** | <ExternalLink href="/docs/openapi/nftTransfer">`POST https://sandbox-okto-gateway.oktostage.com/rpc`</ExternalLink> | `Content-Type: application/json` `Authorization: Bearer <OKTO_AUTH_TOKEN>` | Handles NFT transfers between addresses. |
| **Raw Transaction** | <ExternalLink href="/docs/openapi/evmRawTransaction">`POST https://sandbox-okto-gateway.oktostage.com/rpc`</ExternalLink> | `Content-Type: application/json` `Authorization: Bearer <OKTO_AUTH_TOKEN>` | Handles custom EVM transactions with specified parameters. |
| **Swap Execute** | <ExternalLink href="/docs/openapi/swap/swapIntentExecute">`POST https://sandbox-okto-gateway.oktostage.com/rpc`</ExternalLink> | `Content-Type: application/json` `Authorization: Bearer <OKTO_AUTH_TOKEN>` | Executes token swap operations. |
| **Sign Message** | <ExternalLink href="/docs/openapi/signMessage">`POST https://sandbox-okto-gateway.oktostage.com/rpc`</ExternalLink> | `Content-Type: application/json` `Authorization: Bearer <OKTO_AUTH_TOKEN>` | Signs messages following the EIP191 standard. |
| **Sign Typed Data** | <ExternalLink href="/docs/openapi/signTypedData">`POST https://sandbox-okto-gateway.oktostage.com/rpc`</ExternalLink> | `Content-Type: application/json` `Authorization: Bearer <OKTO_AUTH_TOKEN>` | Signs typed data following the EIP712 standard. |

### REST APIs

REST APIs communicate with the Okto Gateway via a BFF service and are designed for explorer methods.

| **Name** | **API Endpoint** | **Headers** | **Description** |
|----------|------------------|-------------|-----------------|
| **Verify Session** | <ExternalLink href="/docs/openapi/explorer/user-session-verify">`GET /api/oc/v1/verify-session`</ExternalLink> | `Authorization: Bearer <OKTO_AUTH_TOKEN>` | Verify the session of the user. |
| **Send Email OTP** | <ExternalLink href="/docs/openapi/explorer/email-send-otp">`POST /api/oc/v1/authenticate/email`</ExternalLink> | `x-api-key: <API_KEY>` `Content-Type: application/json` | Send OTP to user's email. |
| **Verify Email OTP** | <ExternalLink href="/docs/openapi/explorer/email-verify-otp">`POST /api/oc/v1/authenticate/email/verify`</ExternalLink> | `x-api-key: <API_KEY>` `Content-Type: application/json` | Verify OTP sent to email. |
| **Send WhatsApp OTP** | <ExternalLink href="/docs/openapi/explorer/whatsapp-send-otp">`POST /api/oc/v1/authenticate/whatsapp`</ExternalLink> | `x-api-key: <API_KEY>` `Content-Type: application/json` | Send OTP to user's WhatsApp. |
| **Resend WhatsApp OTP** | <ExternalLink href="/docs/openapi/explorer/whatsapp-resend-otp">`POST /api/oc/v1/authenticate/whatsapp/`</ExternalLink> | `x-api-key: <API_KEY>` `Content-Type: application/json` | Resend OTP to user's WhatsApp. |
| **Verify WhatsApp OTP** | <ExternalLink href="/docs/openapi/explorer/verify-whatsapp-otp">`POST /api/oc/v1/authenticate/whatsapp/verify`</ExternalLink> | `x-api-key: <API_KEY>` `Content-Type: application/json` | Verify OTP sent to WhatsApp. |
| **Get Wallets** | <ExternalLink href="/docs/openapi/explorer/get-wallets">`GET /api/oc/v1/wallets`</ExternalLink> | `Authorization: Bearer <OKTO_AUTH_TOKEN>` | Fetch all the wallets created by the user. |
| **Get Supported Networks** | <ExternalLink href="/docs/openapi/explorer/get-supported-networks">`GET /api/oc/v1/supported/networks`</ExternalLink> | `Authorization: Bearer <OKTO_AUTH_TOKEN>` | Get an array of all the whitelisted networks on your API key. |
| **Get Supported Tokens** | <ExternalLink href="/docs/openapi/explorer/get-supported-tokens">`GET /api/oc/v1/supported/tokens`</ExternalLink> | `Authorization: Bearer <OKTO_AUTH_TOKEN>` | Get an array of all the whitelisted tokens on your API key. |
| **Get Portfolio** | <ExternalLink href="/docs/openapi/explorer/get-portfolio">`GET /api/oc/v1/aggregated-portfolio`</ExternalLink> | `Authorization: Bearer <OKTO_AUTH_TOKEN>` | Get the portfolio data of the user. |
| **Get Portfolio Activity** | <ExternalLink href="/docs/openapi/explorer/get-portfolio-activity">`GET /api/oc/v1/portfolio/activity`</ExternalLink> | `Authorization: Bearer <OKTO_AUTH_TOKEN>` | Get user's portfolio activity. |
| **Get NFT Portfolio** | <ExternalLink href="/docs/openapi/explorer/get-nft-portfolio">`GET /api/oc/v1/portfolio/nft`</ExternalLink> | `Authorization: Bearer <OKTO_AUTH_TOKEN>` | Get the details of all the NFTs. |
| **Get Order History** | <ExternalLink href="/docs/openapi/explorer/get-order-history">`GET /api/oc/v1/orders`</ExternalLink> | `Authorization: Bearer <OKTO_AUTH_TOKEN>` | Fetch all the orders of the user. |
| **Token Transfer Estimate** | <ExternalLink href="/docs/openapi/tokenTransferEstimate">`POST /api/oc/v1/estimate`</ExternalLink> | `Authorization: Bearer <OKTO_AUTH_TOKEN>` `Content-Type: application/json` | Estimate gas for token transfers. |
| **NFT Transfer Estimate** | <ExternalLink href="/docs/openapi/nftTransferEstimate">`POST /api/oc/v1/estimate`</ExternalLink> | `Authorization: Bearer <OKTO_AUTH_TOKEN>` `Content-Type: application/json` | Estimate gas for NFT transfers. |
| **Raw Transaction Estimate** | <ExternalLink href="/docs/openapi/evmRawTransactionEstimate">`POST /api/oc/v1/estimate`</ExternalLink> | `Authorization: Bearer <OKTO_AUTH_TOKEN>` `Content-Type: application/json` | Estimate gas for raw EVM transactions. |
| **Swap Intent Estimate** | <ExternalLink href="/docs/openapi/swap/swapIntentEstimate">`POST /api/oc/v1/estimate`</ExternalLink> | `Authorization: Bearer <OKTO_AUTH_TOKEN>` `Content-Type: application/json` | Estimate outcome of a swap intent. |
| **Read Contract Data (EVM)** | <ExternalLink href="/docs/openapi/explorer/read-contract-data-e-v-m">`POST /api/oc/v1/readContractData`</ExternalLink> | `Authorization: Bearer <OKTO_AUTH_TOKEN>` `Content-Type: application/json` | Read data from EVM smart contracts. |
| **Read Contract Data (Aptos)** | <ExternalLink href="/docs/openapi/explorer/read-contract-data-aptos">`POST /api/oc/v1/readContractData/`</ExternalLink> | `Authorization: Bearer <OKTO_AUTH_TOKEN>` `Content-Type: application/json` | Read data from Aptos smart contracts. |
