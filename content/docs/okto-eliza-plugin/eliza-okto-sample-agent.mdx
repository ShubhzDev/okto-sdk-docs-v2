## Quick Start Guide

Get started quickly with our pre-configured example that demonstrates the Okto Eliza Plugin in action. This implementation includes authentication setup, basic Web3 operations, and example conversations.

<CollapsibleCallout type="tip" title="What's Included">
- Pre-configured Okto Plugin integration
- Google OAuth authentication setup
- Example AI agent implementation
- Sample Web3 operations and commands
- Basic conversation flows
</CollapsibleCallout>

<Steps>

<Step>
### Prerequisites

Before running the sample application, ensure you have:
- **Node.js:** v23+ (required) - <ExternalLink href="https://nodejs.org/">Download</ExternalLink>
- **Package Manager:** pnpm - <ExternalLink href="https://pnpm.io/installation">Install Guide</ExternalLink>
- **Okto Account:** Active account with API credentials
- **LLM Provider:** Access to OpenAI, Anthropic, or other supported providers
</Step>

<Step>
### Setup & Installation

Clone the repository and set up the initial project structure:

```bash title="bash"
# Clone the repository
git clone https://github.com/okto-hq/okto-eliza-ai-agent-example

# Navigate and setup
cd okto-eliza-ai-agent-example
cp .env.sample .env

# Install dependencies
pnpm install
```
</Step>

<Step>
### Configuration

Configure the application settings and character parameters:

1. Edit `.env` with your credentials
2. Configure required environment variables
3. Review and update application settings as needed

<CollapsibleCallout type="tip" title="Customizing the LLM Provider">
Currently, this sample agent uses Groq as the LLM model provider. If you prefer a different provider:
1. Open `characters/okto.character.json`
2. Update the LLM provider configuration
3. Adjust character parameters as needed
</CollapsibleCallout>
</Step>

<Step>
### Launch

Start the application and begin testing:

```bash title="bash"
pnpm run start
```

Try these example commands to test the integration:
- "Show me my wallet balance"
- "What chains are supported?"
- "Help me transfer some tokens"
</Step>

</Steps>

## Available Actions

The sample agent demonstrates the following Okto Eliza Plugin actions:

| Action | Description | GitHub Source |
|---------|-------------|---------------|
| `OKTO_GET_PORTFOLIO` | Get user's token portfolio | <ExternalLink href="https://github.com/okto-hq/eliza-plugin/blob/main/src/actions/getPortfolioAction.ts">Source</ExternalLink> |
| `OKTO_GET_ACCOUNT` | Get wallet accounts | <ExternalLink href="https://github.com/okto-hq/eliza-plugin/blob/main/src/actions/getAccountAction.ts">Source</ExternalLink> |
| `OKTO_GET_CHAINS` | List supported networks | <ExternalLink href="https://github.com/okto-hq/eliza-plugin/blob/main/src/actions/getChainAction.ts">Source</ExternalLink> |
| `OKTO_GET_NFT_COLLECTIONS` | Get NFT collections | <ExternalLink href="https://github.com/okto-hq/eliza-plugin/blob/main/src/actions/getNftCollectionsAction.ts">Source</ExternalLink> |
| `OKTO_GET_ORDERS_HISTORY` | View transaction history | <ExternalLink href="https://github.com/okto-hq/eliza-plugin/blob/main/src/actions/getOrdersHistoryAction.ts">Source</ExternalLink> |
| `OKTO_GET_PORTFOLIO_NFT` | Get NFT portfolio | <ExternalLink href="https://github.com/okto-hq/eliza-plugin/blob/main/src/actions/getPortfolioNftAction.ts">Source</ExternalLink> |
| `OKTO_GET_TOKENS` | List supported tokens | <ExternalLink href="https://github.com/okto-hq/eliza-plugin/blob/main/src/actions/getTokensAction.ts">Source</ExternalLink> |
